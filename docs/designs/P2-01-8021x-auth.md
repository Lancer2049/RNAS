# P2-01: 802.1X 认证 (EAP-PEAP/EAP-TLS) 详细设计

## 用户故事
> 作为企业网络工程师，我需要 RNAS 仿真 802.1X 认证（交换机端口认证/WiFi WPA2-Enterprise 场景），验证 RADIUS 服务器对 EAP-PEAP 和 EAP-TLS 的处理逻辑，包括证书验证、VLAN 分配、ACL 下发等企业级特性。

## 当前状态
- ✅ PPPoE/PPTP/L2TP/SSTP/IPoE 的 PAP 认证
- ✅ RADIUS Access-Request/Accept/Reject
- ✅ FreeRADIUS 作为后端
- ❌ 无 802.1X 支持
- ❌ 无 EAP 协议栈
- ❌ 无 hostapd 集成

## 技术方案

### 方案选择

#### 方案 A：hostapd + accel-ppp（推荐）
利用 hostapd 作为 802.1X authenticator，将 EAP 转换为 RADIUS 请求发到 VM2。
```
VM1 (Supplicant)          VM3 (Authenticator)         VM2 (RADIUS)
  wpa_supplicant             hostapd                   FreeRADIUS
      |                          |                         |
      |-- EAPOL-Start ---------->|                         |
      |                          |-- Access-Request ------>|
      |                          |   (EAP-Message=...)     |
      |                          |<-- Access-Challenge ----|
      |<-- EAP-Request ----------|                         |
      |-- EAP-Response -------->|                          |
      |                          |-- Access-Request ------>|
      |                          |<-- Access-Accept -------|
      |<-- EAP-Success ---------|   (Tunnel-Private-Group) |
      |                          |                         |
```

优点：标准实现，全协议支持，VLAN/ACL 下发  
缺点：需要 hostapd + wpa_supplicant 包

#### 方案 B：accel-ppp IPoE + 802.1X 模块
accel-ppp 有内置 dot1x 模块，可直接作为 authenticator
优点：单进程，与现有集成好
缺点：依赖编译选项（当前未启用）

**推荐方案 A**：hostapd 成熟稳定，配置灵活。

### 配置模板

`/etc/rnas/wireless.d/dot1x.conf`：
```ini
[dot1x]
enabled = yes
interface = ens33
auth_server = 192.168.0.202
auth_port = 1812
auth_secret = testing123
nas_identifier = rnas-dot1x
eap_methods = peap,tls,md5
private_key = /etc/rnas/ssl/dot1x.key
certificate = /etc/rnas/ssl/dot1x.crt
ca_cert = /etc/rnas/ssl/ca.crt
```

### hostapd 配置生成器

rnas-config 输出：
```
interface=ens33
driver=wired
ieee8021x=1
auth_server_addr=192.168.0.202
auth_server_port=1812
auth_server_shared_secret=testing123
eap_server=0
eap_user_file=/etc/rnas/dot1x-users
ca_cert=/etc/rnas/ssl/ca.crt
server_cert=/etc/rnas/ssl/dot1x.crt
private_key=/etc/rnas/ssl/dot1x.key
```

### 测试场景

| 场景 | 说明 | 验证点 |
|------|------|--------|
| EAP-PEAP 成功 | wpa_supplicant 正确证书 + 密码 | Access-Accept |
| EAP-PEAP 密码错误 | 错误密码 | Access-Reject |
| EAP-TLS 证书认证 | 双向证书验证 | Access-Accept + 客户端证书验证 |
| EAP-TLS 证书过期 | 过期客户端证书 | Access-Reject |
| VLAN 分配 | RADIUS 返回 Tunnel-Private-Group-ID | 客户端分配到指定 VLAN |
| 会话超时 | RADIUS 返回 Session-Timeout | 到达超时后自动断开 |

### 测试拓扑
```
VM1: wpa_supplicant (802.1X supplicant)
     ↓ EAPOL over LAN
VM3: hostapd (authenticator) → RADIUS → VM2
     Web Dashboard 显示 802.1X 会话
```

### 实现步骤

| 步骤 | 内容 | 产出 |
|------|------|------|
| 1 | 安装 hostapd + wpa_supplicant | 包依赖 |
| 2 | 生成自签名证书 (CA + server + client) | PKI 文件 |
| 3 | 编写 dot1x 配置模板 | `/etc/rnas/wireless.d/dot1x.conf` |
| 4 | rnas-config 生成 hostapd.conf | `generate_dot1x()` |
| 5 | 配置 FreeRADIUS eap 模块 | EAP-PEAP + EAP-TLS |
| 6 | 端到端测试 | 6 种场景 |
| 7 | Web UI 会话显示 | 扩展 SessionsTable |

### 证书管理

内置工具自动生成测试证书：
```bash
rnas-dot1x init    # 生成 CA + server cert
rnas-dot1x client  # 生成客户端证书
rnas-dot1x revoke  # 吊销客户端证书
```

### 验收标准

| # | 标准 | 验证方法 |
|---|------|---------|
| 1 | EAP-PEAP 认证成功 | wpa_supplicant 连接成功 |
| 2 | RADIUS Access-Accept 包含 EAP-Success | 抓包 + radacct 记录 |
| 3 | EAP-TLS 双向证书认证 | 客户端和服务器证书均验证 |
| 4 | VLAN 分配生效 | 客户端接口获得 Tunnel-Private-Group 指定的 VLAN |
| 5 | 错误密码被拒绝 | wpa_supplicant 报告认证失败 |
| 6 | Web UI 显示 802.1X 会话 | Playwright 测试 |

### 依赖
- VM3: hostapd, openssl
- VM1: wpa_supplicant
- VM2: FreeRADIUS eap 模块（需配置）
- 无需额外内核模块

### 预估工时
10-14 小时（安装 1h + PKI 2h + 配置模板 2h + 生成器 2h + FreeRADIUS 2h + 测试 3h + UI 2h）
